# 店舗管理システム（store-tool）

本部と店舗の業務指示・回答を一元化し、  
進捗と実行率を可視化する **本部・店舗向け業務指示ツール**です。

紙やメールではなく Web で指示を配信し、  
店舗の回答・完了を一覧で把握できます。

## できること

### 本部向け

- 業務指示・ルーティンの作成・配信（全店舗／グループ／指定店舗）
- 店舗ごとの回答・進捗確認、締切管理
- 要望・リクエストの受付、返信・完了処理
- 実行率・前日比・前週比のダッシュボード
- ユーザー・店舗・店舗グループの管理
- よくある問い合わせ（QA）の登録・管理
- 監査ログ、バックアップ・リストア
- メッセージ機能（設定でオン/オフ可能）

### 店舗向け

- 自店舗あての業務指示・ルーティンの一覧・詳細確認
- 回答・完了の送信（権限に応じて）
- 要望・リクエストの送信、本部返信の確認
- よくある問い合わせの閲覧
- 本部とのメッセージ（設定で有効時のみ）
- スマホ縦画面でも操作しやすい画面

## 特徴

- **閉域ネットワーク前提**
- **社内サーバー設置型**
- 店舗コード・社員番号・パスワードでログイン（JWT）
- 本部と店舗でロール分け（店舗はマネージャー／スタッフ、閲覧のみ設定可）
- DB は SQLite（1ファイル・軽量・運用が簡単）
- 静的 HTML + vanilla JS（SPA なし・挙動が分かりやすい）
- 画像添付対応（業務指示・要望・リクエスト・メッセージ）

## 想定利用シーン

- 小規模〜中規模のチェーン・FC
- 本部と複数店舗で業務指示を回している組織
- 紙・メール・チャットが混在して進捗が見えにくい現場
- 実行率や店舗別の状況をダッシュボードで把握したい本部

---

## 導入方法（概要）

1. 社内サーバーにファイルを配置
2. Node.js（18以上）をインストール
3. `npm install` のあと `node src/index.js` で起動（Windows では `start.bat` も利用可）
4. ブラウザで社内 IP にアクセス

※ 詳細な手順は導入時に案内します
